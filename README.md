This video demonstrates a real-time KPI Evaluation and Alert Notifier System built using Apache NiFi. The project focuses on designing a scalable data flow architecture to process KPI data, evaluate performance thresholds, and trigger alerts efficiently.

Key Features:

KPI evaluation using custom thresholds (latency, throughput, packet loss, handover, etc.)
Automated alert generation for abnormal conditions
Stream processing and routing using Apache NiFi processors
Scalable and modular data flow design

Technologies Used:

Apache NiFi (Data Flow Management)
Kafka (for streaming, if used)
SQL / QueryRecord for KPI filtering
JSON/CSV data processing

Use Case:
This system is highly relevant for telecommunication networks (4G/5G) where continuous KPI monitoring is critical to ensure network performance and reliability.

What you’ll learn:

How to design a real-time data pipeline using Apache NiFi
How to process and split KPI data streams
How to implement alerting logic in data flows
Best practices for building scalable monitoring systems
